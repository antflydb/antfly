# OpenAPI

This repository keeps generated Zig OpenAPI code checked in. Normal builds import
the generated modules from stable paths and do not run `openapi-zig` as part of
the critical build path.

## Common Commands

Regenerate OpenAPI specs and Zig modules:

```sh
make generate
```

Run the OpenAPI consistency check used by CI:

```sh
make openapi-check
```

Run only the Zig code generator step:

```sh
zig build regen-openapi
```

## Source Specs

Repo-local specs:

- `../specs/openapi/antfly/public.yaml`: bundled public Antfly API spec
- `../specs/openapi/antfly/metadata.yaml`: Antfly metadata/table API
- `../specs/openapi/antfly/usermgr.yaml`: Antfly user-management API
- `../specs/openapi/termite/api.yaml`: Termite public API
- `openapi.yaml`: compatibility copy of the bundled public Antfly API spec
- `specs/openai-openapi.yaml`: vendored OpenAI API schema used for types

Some shared provider/config schemas are still read from the sibling
`../antfly` checkout when regenerating. The checked-in generated Zig means a
normal build does not need that sibling checkout, but `make generate` currently
does.

## Generated Modules

Generated Antfly and shared API modules live under:

```text
pkg/antfly/src/openapi/generated/
```

The generated Termite API module lives under:

```text
pkg/termite/src/api/generated/termite_api/
```

Generated files start with:

```zig
// Code generated by openapi-zig. DO NOT EDIT.
```

Do not edit generated files by hand. Update the source spec or generator, then
run `make generate`.

## Build Behavior

Top-level builds import generated modules directly from the checked-in
directories. This avoids repeatedly converting YAML to JSON and running
`openapi-zig` during normal builds.

Standalone Termite builds also use the checked-in `termite_api` module by
default. For experiments, the old dynamic codegen path remains available:

```sh
cd pkg/termite
zig build -Dtermite-openapi-spec=../../../specs/openapi/termite/api.yaml
```

## Updating An API

1. Edit the owning OpenAPI YAML file.
2. Run `make generate`.
3. Review both the spec diff and generated Zig diff.
4. Run a focused compile or test for the affected API.

Useful checks:

```sh
zig build root-test
zig build openapi-root-check
cd pkg/termite && zig build test
```

`openapi-root-check` verifies that the bundled `openapi.yaml` matches the
modular Antfly specs.

## Runtime Paths

Antfly public APIs are served under:

```text
/api/v1/*
```

Termite public APIs are served under:

```text
/ml/v1/*
```

Termite keeps `/embed` and `/embeddings` as aliases over the same
`EmbedRequest` / `EmbedResponse` contract.
