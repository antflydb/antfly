# Group IDs

Antfly uses `u64` group IDs for Raft groups. The ID space is shared by metadata
groups and data groups, so the namespace must be partitioned explicitly.

## Namespace

- System groups set the high bit: `group_id & (1 << 63) != 0`.
- Data groups clear the high bit and are never zero.
- The main metadata group is `0x8000000000000001`.

Data group IDs are generated by hashing table/range inputs and clearing the
system bit. User-provided data group IDs must also be below `1 << 63`.

## Rationale

The Raft host routes a group to the metadata state machine when its group ID
equals the configured metadata group ID; all other groups route to data state
machines. If a data group can collide with the metadata group, data writes may
be interpreted as metadata commands. Reserving the high-bit half of the ID space
removes that class of collision while keeping a large data namespace.

## Compatibility

Go currently uses metadata group `1` internally and does not expose it as a CLI
option. The Zig implementation uses a Zig-specific system ID for the main
metadata group so data groups can retain the low-bit namespace.
