# openapi-zig: OpenAPI 3.0.x to Zig Code Generator

## Context

We want an oapi-codegen equivalent for Zig — a tool that reads OpenAPI 3.0.x JSON specs and generates typed Zig client/server code targeting httpx.zig. The antfly2 OpenAPI specs serve as the primary test case, covering complex patterns: `oneOf`/`allOf` with discriminators, cross-file `$ref`, enums, nullable fields, `additionalProperties`, SSE streaming, and multiple auth schemes.

The tool lives at `/Users/ajroetker/go/src/github.com/antflydb/openapi-zig/` and builds with `~/bin/zig` (0.16.0-dev).

## Key Design Decisions

### Input: JSON only (YAML→JSON is external)
Use `redocly bundle spec.yaml -o spec.json` to convert+bundle YAML specs into a single JSON file with all `$ref`s resolved. This avoids implementing a YAML parser in Zig and sidesteps cross-file resolution complexity for Phase 1.

### Parsing strategy: Two-phase
1. Parse JSON into `std.json.Value` (dynamic)
2. Walk the dynamic tree into typed `OpenAPI` AST nodes

This avoids fighting `std.json.parseFromSlice` with OpenAPI's polymorphic structures (schemas can be inline objects, `$ref` strings, or arrays of either).

### Code emission: Direct string writing
Use a `SourceWriter` with indent tracking — no template engine needed. Generated files get a `// Code generated by openapi-zig. DO NOT EDIT.` header.

## Type Mapping

| OpenAPI | Zig |
|---------|-----|
| `string` | `[]const u8` |
| `string` + `format: date-time` | `[]const u8` |
| `string` + `enum: [...]` | `enum { ... }` |
| `integer` / `int32` | `i32` |
| `integer` + `int64` | `i64` |
| `number` / `double` | `f64` |
| `number` + `float` | `f32` |
| `boolean` | `bool` |
| `array` of T | `[]const T` |
| `object` with `properties` | `struct { ... }` |
| `object` + `additionalProperties: true` | `std.json.Value` |
| `object` + `additionalProperties: {schema}` | `std.json.ArrayHashMap(T)` |
| `nullable: true` | `?T` |
| optional (not in `required`) | `?T = null` |
| `oneOf` + discriminator | `union(enum) { ... }` + custom `jsonParse` |
| `allOf` | flattened `struct` merging all properties |
| `anyOf` | `std.json.Value` (runtime typed) |
| `$ref: "#/components/schemas/Foo"` | `Foo` (by name) |

### Client (`client.zig`)

```zig
const std = @import("std");
const httpx = @import("httpx");
const types = @import("types.zig");

pub fn ApiResponse(comptime T: type) type {
    return struct {
        status_code: u16,
        data: ?std.json.Parsed(T) = null,
        err: ?std.json.Parsed(types.Error) = null,

        pub fn deinit(self: *@This()) void { ... }
    };
}

pub const Client = struct {
    http: *httpx.Client,
    base_url: []const u8,
    allocator: std.mem.Allocator,
    auth_header: ?[2][]const u8 = null,

    pub fn init(allocator: std.mem.Allocator, http: *httpx.Client, base_url: []const u8) @This() { ... }

    /// GET /status
    pub fn getStatus(self: *@This()) !ApiResponse(types.ClusterStatus) { ... }

    /// PUT /secrets/{key}
    pub fn putSecret(self: *@This(), key: []const u8, body: types.SecretWriteRequest) !ApiResponse(types.SecretEntry) {
        const url = try std.fmt.allocPrint(self.allocator, "{s}/secrets/{s}", .{ self.base_url, key });
        defer self.allocator.free(url);
        const json_body = try httpx.json.Json.stringify(self.allocator, body);
        defer self.allocator.free(json_body);
        var resp = try self.http.put(url, .{ .json = json_body, .headers = self.authHeaders() });
        return ApiResponse(types.SecretEntry).fromResponse(self.allocator, &resp);
    }
};
```

### Server (`server.zig`)

Uses comptime generics — user provides an impl struct:

```zig
pub fn ServerRouter(comptime Impl: type) type {
    return struct {
        impl: *Impl,

        pub fn register(self: @This(), server: *httpx.Server) !void {
            // Store impl pointer, register routes with wrapper handlers
            try server.get("/status", getStatusHandler);
            try server.put("/secrets/:key", putSecretHandler);
        }
    };
}
```

Handler wrappers extract path params, parse request bodies, call `impl.methodName()`, and serialize responses via `ctx.json()`.

## Build Integration

### As standalone CLI
```bash
npx @redocly/cli bundle api.yaml -o api.json
~/bin/zig build run -- --spec api.json --output generated/ --package my_api
```

### As build.zig dependency
```zig
const openapi_dep = b.dependency("openapi_zig", .{});
const codegen = b.addRunArtifact(openapi_dep.artifact("openapi-zig"));
codegen.addArgs(&.{ "--spec" });
codegen.addFileArg(b.path("api.json"));
codegen.addArgs(&.{ "--package", "my_api", "--output-dir" });
const gen = codegen.addOutputDirectoryArg("my_api");

const api_mod = b.addModule("my_api", .{ .root_source_file = gen.path(b, "root.zig") });
api_mod.addImport("httpx", httpx_mod);
```

## Implementation Phases

### Phase 1: Foundation — AST + Type Codegen
**Files:** `build.zig`, `build.zig.zon`, `openapi.zig`, `parser.zig`, `naming.zig`, `writer.zig`, `codegen_types.zig`, `codegen.zig`, `main.zig`

1. Project skeleton with httpx.zig as path dependency
2. OpenAPI AST types (`OpenApiDoc`, `Schema`, `PathItem`, `Operation`, etc.)
3. JSON→AST parser (walk `std.json.Value` tree)
4. Naming utilities (PascalCase for types, snake_case for fields, escape `type`→`type_`)
5. `SourceWriter` with indentation tracking
6. Type codegen: primitives, string enums, structs with required/optional fields, nullable
7. CLI: `--spec`, `--output`, `--package` args
8. Test with hand-crafted fixture JSON

### Phase 2: Client Generation
**Files:** `codegen_client.zig`, extend `codegen.zig`

1. Generate `ApiClient` struct with method per `operationId`
2. Path parameter interpolation via `std.fmt.allocPrint`
3. Query parameters as optional struct argument
4. Request body serialization (JSON)
5. `ApiResponse(T)` wrapper with success/error handling
6. Auth header injection (Bearer, Basic, ApiKey)

### Phase 3: Complex Types + $ref
**Files:** `resolver.zig`, extend `codegen_types.zig`

1. Local `$ref` resolution (`#/components/schemas/Foo`)
2. `allOf` → merged struct (flatten all properties)
3. `oneOf` + discriminator → `union(enum)` with custom `jsonParse`
4. `additionalProperties` → `std.json.Value` or typed map
5. Nested inline objects → anonymous inner structs
6. Topological sort for type emission order
7. Test with bundled antfly2 `openapi.yaml`

### Phase 4: Server Generation
**Files:** `codegen_server.zig`, extend `codegen.zig`

1. `ServerRouter(Impl)` comptime generic
2. Wrapper handlers: path param extraction, body parsing, response serialization
3. Query param extraction helpers
4. Error response helpers (400, 404, 500)

### Phase 5: Advanced Features
1. SSE streaming endpoints (return raw `httpx.Response`)
2. Binary response support (`application/octet-stream`)
3. `anyOf` fallback to `std.json.Value`
4. Config file support (generate subsets, import mappings)
5. Cross-file `$ref` resolution (non-bundled specs)
