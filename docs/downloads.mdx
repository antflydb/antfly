## Download Antfly

### Early-Access Release

Antfly is currently in pre-alpha development.
We are actively working on features and improvements, and we welcome your feedback.

Antfly ships with bundled [ONNX Runtime](https://onnxruntime.ai/) and [XLA](https://openxla.org/) backends for fast local ML inference. This allows Antfly to generate embeddings, rerank results, and chunk text directly without requiring an external service like Ollama.

or tap homebrew-antfly and install separately:

```bash
brew tap antflydb/antfly
brew install --cask antfly
```

This installs `antfly` with bundled ONNX Runtime and XLA libraries for local ML inference. CLI commands are available directly as `antfly <command>` (e.g., `antfly query`, `antfly table create`).

  <TabsContent value="macos-manual">
<Steps>
### Install using the install script (recommended)

```bash
curl -fsSL https://releases.antfly.io/antfly/latest/install.sh | sh -s -- --omni
```

Or install a specific version:

```bash
curl -fsSL https://releases.antfly.io/antfly/latest/install.sh | sh -s -- --omni {ANTFLY_VERSION}
```

### Or manually extract the binaries

```bash
tar -xzf antfly-omni_{ANTFLY_VERSION}_Darwin_arm64.tar.gz
```

### Make the binaries executable

```bash
chmod +x antfly
```

### Remove the quarantine attribute

```bash
xattr -dr com.apple.quarantine antfly lib/
```

### Ensure the binaries and libraries are accessible

The `lib/` directory contains the bundled ONNX Runtime and XLA libraries. Keep it alongside the `antfly` binary, or set `DYLD_LIBRARY_PATH`:

```bash
export DYLD_LIBRARY_PATH=$(pwd)/lib:$DYLD_LIBRARY_PATH
export PATH=$PATH:$(pwd)
```
</Steps>

  <TabsContent value="docker">
<Steps>
### Create an `entrypoint.sh`

```bash
#!/bin/bash

# Start Ollama in the background.
/bin/ollama serve &
# Record Process ID.
pid=$!

# Pause for Ollama to start.
sleep 5

echo "🔴 Retrieve ALL-MINILM model..."
ollama pull all-minilm
echo "🟢 Done!"

# Wait for Ollama process to finish.
wait $pid
```

### Create a `config.json`

```json
{
  "metadata": {
    "orchestration_urls": { "1": "http://127.0.0.1:12377" }
  },
  "replication_factor": 1,
  "max_shards_per_table": 4
}
```

### Create a `docker-compose.yml`

```yaml
services:
  antfly:
    image: ghcr.io/antflydb/antfly:latest
    command: ["swarm"]
    ports:
      - "8080:8080" # Public API port
      - "9090:9090" # MCP port
    volumes:
      - ./antfly-data:/antflydb
      - ./config.json:/config.json
    container_name: antfly-swarm
    environment:
      - ANTFLY_LOG_LEVEL=info
    restart: always
    networks:
      - antfly-network

  ollama:
    image: ollama/ollama:latest
    ports:
      - "11434:11434"
    volumes:
      - ./ollama-data:/root/.ollama
      - ./entrypoint.sh:/entrypoint.sh
    container_name: ollama
    pull_policy: always
    tty: true
    restart: always
    entrypoint: ["/usr/bin/bash", "/entrypoint.sh"]
    networks:
      - antfly-network

networks:
  antfly-network:
    driver: bridge

volumes:
  antfly-data:
  ollama-data:
```

### Run `docker-compose up -d`
