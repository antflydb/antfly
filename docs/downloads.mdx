## Download Antfly

### Early-Access Release

Antfly is currently in pre-alpha development.
We are actively working on features and improvements, and we welcome your feedback.

Antfly now ships the native Zig runtime as the default `antfly` binary. The previous Go/omni runtime remains available as `antfly-go` for old data directories, rollback, and portable backup creation before restoring into a fresh Zig data directory.

  <TabsContent value="brew">
```bash
brew install antflydb/taps/antfly
```

or tap homebrew-taps and install separately:

```bash
brew tap antflydb/taps
brew install antfly
```

This installs the native Zig runtime as `antfly`. CLI commands are available directly as `antfly <command>` (e.g., `antfly query`, `antfly table create`).

To run Antfly as a local service:

```bash
brew services start antfly
```

When migrating from the Go/omni runtime to the Zig runtime, do not reuse the old data directory in place. Create a portable backup with the previous Go/omni binary, start the Zig runtime with a fresh data directory, then restore the backup.

To keep using the previous Go/omni runtime or create that backup:

```bash
brew install antflydb/taps/antfly-go
antfly-go --help
```

  <TabsContent value="macos-manual">
<Steps>
### Install using the install script (recommended)

```bash
curl -fsSL https://releases.antfly.io/antfly/latest/install.sh | sh
```

Or install a specific version:

```bash
curl -fsSL https://releases.antfly.io/antfly/latest/install.sh | sh -s -- {ANTFLY_VERSION}
```

### Or manually extract the binaries

```bash
tar -xzf antfly_{ANTFLY_VERSION}_Darwin_arm64.tar.gz
```

### Make the binaries executable

```bash
chmod +x antfly
```

### Remove the quarantine attribute

```bash
xattr -dr com.apple.quarantine antfly
```

### Ensure the binary is accessible

Add the extracted directory to your `PATH`:

```bash
export PATH=$PATH:$(pwd)
```

### Go/omni fallback

Use `--go` to install the previous runtime as `antfly-go`:

```bash
curl -fsSL https://releases.antfly.io/antfly/latest/install.sh | sh -s -- --go
```
</Steps>

  <TabsContent value="docker">
<Steps>
### Create an `entrypoint.sh`

```bash
#!/bin/bash

# Start Ollama in the background.
/bin/ollama serve &
# Record Process ID.
pid=$!

# Pause for Ollama to start.
sleep 5

echo "🔴 Retrieve ALL-MINILM model..."
ollama pull all-minilm
echo "🟢 Done!"

# Wait for Ollama process to finish.
wait $pid
```

### Create a `config.json`

```json
{
  "metadata": {
    "orchestration_urls": { "1": "http://127.0.0.1:12377" }
  },
  "replication_factor": 1,
  "max_shards_per_table": 4
}
```

### Create a `docker-compose.yml`

```yaml
services:
  antfly:
    image: ghcr.io/antflydb/antfly:latest
    command: ["swarm"]
    ports:
      - "8080:8080" # Public API port
      - "9090:9090" # MCP port
    volumes:
      - ./antfly-data:/antflydb
      - ./config.json:/config.json
    container_name: antfly-swarm
    environment:
      - ANTFLY_LOG_LEVEL=info
    restart: always
    networks:
      - antfly-network

  ollama:
    image: ollama/ollama:latest
    ports:
      - "11434:11434"
    volumes:
      - ./ollama-data:/root/.ollama
      - ./entrypoint.sh:/entrypoint.sh
    container_name: ollama
    pull_policy: always
    tty: true
    restart: always
    entrypoint: ["/usr/bin/bash", "/entrypoint.sh"]
    networks:
      - antfly-network

networks:
  antfly-network:
    driver: bridge

volumes:
  antfly-data:
  ollama-data:
```

### Run `docker-compose up -d`
