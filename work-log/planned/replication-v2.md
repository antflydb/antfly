# CDC Route-Based Filtering with EvaluateFilter

## Context

Antfly's CDC replication currently supports 1 PG table → 1 Antfly table per replication source. For data warehouse use cases (BigQuery, multi-tenant PG), we need:
- **Fan-out**: one PG table → multiple Antfly tables based on row conditions
- **Filtering**: skip rows that don't match criteria
- **PG-side filtering**: push simple filters to the publication WHERE clause

The approach uses the existing bleve filter query JSON DSL (already used for `filter_query`, `exclusion_query`, joins) as the `where` clause on routes, evaluated in-memory via a new `EvaluateFilter` function — no scorch/bleve index needed.

## Implementation Steps

### Step 1: `EvaluateFilter` function

**New file**: `src/metadata/foreign/evaluate_filter.go`

Mirrors `TranslateFilter` in `filter.go` — same JSON parsing, same key-detection pattern, but evaluates against `map[string]any` instead of emitting SQL.

```go
func EvaluateFilter(filter json.RawMessage, doc map[string]any) (bool, error)
```

- nil/empty filter → `(true, nil)` (match-all)
- Same recursive `evaluate(raw, doc)` dispatcher as `filterCtx.translate`
- Cases: `term`, `match`, `prefix`, `wildcard`, `min`/`max` range, `conjuncts`, `disjuncts`, `must_not`, `query` string, `match_all`
- Missing field in doc → `false` (no error), nil field value → `false`
- Copy `valuesEqual`, `compareOrdered`, `toFloat64` from `src/metadata/join/executor.go` (3 small pure functions, ~50 lines — avoids importing join package with its heavy dependencies)
- Add `matchWildcard(pattern, value string) bool` helper for `*`/`?` glob matching

### Step 2: `EvaluateFilter` tests

**New file**: `src/metadata/foreign/evaluate_filter_test.go`

Parallel structure to `filter_test.go` but verifying boolean results:
- Empty/nil/match_all → true
- Term: exact match, mismatch, numeric cross-type (int term matches float doc value)
- Match: same as term
- Prefix: string prefix match/mismatch
- Wildcard: `*` and `?` patterns
- Range: inclusive/exclusive min/max, partial ranges (min-only, max-only)
- Boolean: conjuncts (AND), disjuncts (OR), must_not (NOT)
- Query string: single `field:value`, `field:v1 OR field:v2`
- Edge cases: missing field, nil value, non-string for prefix/wildcard

### Step 3: `FilterToLiteralSQL` for publication WHERE clauses

**File**: `src/metadata/foreign/filter.go`

Add a new function that translates bleve JSON to SQL with inlined literal values (no `$1` placeholders). This is needed because `CREATE PUBLICATION ... WHERE (expr)` doesn't support prepared statement parameters.

```go
// FilterToLiteralSQL translates a bleve-style filter to a SQL expression with
// literal values inlined, suitable for CREATE PUBLICATION WHERE clauses.
// knownColumns may be nil to skip column validation.
func FilterToLiteralSQL(filter json.RawMessage, knownColumns []ForeignColumn) (string, error)
```

Implementation: reuse `TranslateFilter` with a `PlaceholderFunc` that returns sequentially numbered markers, then replace each marker with the properly quoted literal from args. Use `pgQuoteLiteral` for string values, `fmt.Sprintf` for numerics/bools.

Add tests in `filter_test.go`:
- `TestFilterToLiteralSQL_Term` → `"status" = 'active'`
- `TestFilterToLiteralSQL_Range` → `"age" >= 18 AND "age" <= 65`
- `TestFilterToLiteralSQL_Conjuncts` → `("status" = 'active') AND ("age" >= 18)`

### Step 4: Store config types

**File**: `src/store/table.go`

Add after `ReplicationTransformOp` (line 217):

```go
type ReplicationRouteConfig struct {
    TargetTable string                   `json:"target_table"`
    Where       json.RawMessage          `json:"where,omitempty"`
    KeyTemplate string                   `json:"key_template,omitempty"`
    OnUpdate    []ReplicationTransformOp `json:"on_update,omitempty"`
    OnDelete    []ReplicationTransformOp `json:"on_delete,omitempty"`
}
```

Add to `ReplicationSourceConfig`:
- `PublicationFilter json.RawMessage` field — bleve-style filter JSON, translated to SQL for the PG publication
- `Routes []ReplicationRouteConfig` field

Note: `json.RawMessage` works because `store` already imports `json "github.com/antflydb/antfly-go/libaf/json"` which aliases `RawMessage = stdjson.RawMessage`.

### Step 5: OpenAPI schema

**File**: `src/metadata/api.yaml`

Add `ReplicationRoute` schema (after `ReplicationTransformOp`, ~line 4149):
- `target_table`: required string
- `where`: bleve query ref (`$ref: "../../bleve-query-openapi.yaml#/components/schemas/Query"` with `x-go-type: json.RawMessage`)
- `key_template`: optional string (override source-level)
- `on_update`: optional array of `ReplicationTransformOp`
- `on_delete`: optional array of `ReplicationTransformOp`

Add to `ReplicationSource` properties:
- `publication_filter`: bleve query JSON (same `$ref` + `x-go-type: json.RawMessage` pattern as `filter_query`). Translated to SQL via `FilterToLiteralSQL` for the PG publication WHERE clause.
- `routes`: array of `ReplicationRoute`

### Step 6: Regenerate

Run `make generate` to regenerate `api.gen.go` with new `ReplicationRoute` struct and updated `ReplicationSource`.

### Step 7: Replication config + route fan-out

**File**: `src/metadata/foreign/replication.go`

Add route config type:
```go
type ReplicationRouteConfig struct {
    TargetTable string
    Where       json.RawMessage
    KeyTemplate string
    OnUpdate    []store.ReplicationTransformOp
    OnDelete    []store.ReplicationTransformOp
}
```

Add to `ReplicationConfig`:
- `PublicationFilter json.RawMessage` — bleve JSON, translated to SQL at publication creation time
- `Routes []ReplicationRouteConfig`

Modify `ensurePublication` (line 191-195): when `PublicationFilter` is set, call `FilterToLiteralSQL(w.config.PublicationFilter, nil)` to translate bleve JSON to SQL, then append ` WHERE (<sql>)` to the CREATE PUBLICATION statement.

Modify `processDataChange` (line 372): if `len(w.config.Routes) > 0`, delegate to new `processDataChangeRouted`. Otherwise existing logic unchanged.

New `processDataChangeRouted(ctx, rel, row)`:
- For each route: evaluate `EvaluateFilter(route.Where, row)`, skip if no match
- Use route's `KeyTemplate` (fallback to source-level), route's `OnUpdate` (fallback to auto)
- Call `transformFunc(ctx, route.TargetTable, key, ops, true)`
- Aggregate errors with `errors.Join`

Modify `processDelete` (line 403): same pattern — delegate to `processDeleteRouted` when routes present.

New `processDeleteRouted(ctx, rel, row)`:
- Same filter evaluation per route
- Handle `$delete_document` and transform resolution per route
- Use `autoOnDelete(route.OnUpdate, rel)` when route has no explicit `OnDelete`

### Step 8: Manager config mapping

**File**: `src/metadata/foreign/replication_manager.go`

In `scanAndStart` (line 180-189): pass `src.PublicationFilter` and map `src.Routes` to `ReplicationRouteConfig` list on the `ReplicationConfig`.

### Step 9: API handler mapping

**File**: `src/metadata/api.go`

In `CreateTable` handler (line 397-429): map `src.Routes` and `src.PublicationFilter` from API types to `store.ReplicationSourceConfig`.

In `replicationSourcesToAPI` (line 1501-1533): map routes and publication_filter back to API response types.

### Step 10: Validation

In `CreateTable` handler, after mapping routes:
- Validate each route has non-empty `target_table`
- Validate route `where` parses correctly (call `EvaluateFilter(where, map[string]any{})` to catch parse errors at creation time)
- Validate `publication_filter` translates to SQL (call `FilterToLiteralSQL(filter, nil)` to catch parse errors at creation time)
- Warn if routes present alongside top-level `on_update`/`on_delete` (top-level transforms ignored when routes are used)
